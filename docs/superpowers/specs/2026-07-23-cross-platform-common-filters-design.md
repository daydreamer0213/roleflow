# 跨平台通用高级筛选设计

## 状态

已确认设计；尚未实现。

## 目标

在 RoleFlow 的通用 Search Plan 中增加可跨平台映射的严格高级筛选：区域和公司规模。BOSS、未来的智联等平台各自把通用条件转换为自己的原生筛选参数。

## 范围

本期包含：

- 城市内区域，可选且按城市保存。
- 公司规模，可选且为一个通用规模档。
- 按“平台 + 城市”读取、缓存和校验可用的原生筛选选项。
- 生成模式中的严格预检、提示和 URL 参数映射。

本期不包含：

- 行业筛选。默认不限制行业；行业要求由继承模式满足。
- 融资阶段、平台特有筛选和任意动态字段。
- 改变继承模式的 URL 保留与仅替换关键词行为。

## 用户体验

1. 用户先选择目标城市。
2. 在“高级筛选”中，为每个已选城市选择零或一个区域；不选代表全市。
3. 用户可选择零或一个公司规模档；不选代表不限制规模。
4. 页面预览完整展示将要应用的条件，例如“上海 · 浦东新区 · 100-499 人 · 10-20K · 1-3 年”。
5. 如果某个已选城市或平台无法精确映射已保存的区域或规模，扫描开始前阻止执行并给出具体条件与平台名称。系统不得删除条件、改用近似条件或静默放宽搜索。

继承模式不显示或应用这些通用高级条件：用户在 BOSS 手工设置的区域、行业、规模等 URL 参数原样继承，RoleFlow 只轮换关键词。

## 多平台继承模式补充

继承模式不是 BOSS 专属能力。每一个已接入平台都可以拥有一份由用户主动保存的“搜索页模板”：用户先在该平台手工设置城市、区域、行业、薪资、经验、公司规模及任何平台特有条件，再让 RoleFlow 验证并保存该平台的规范化搜索上下文。

模板必须带有平台归属；BOSS 模板不能用于智联，智联模板也不能被解释成 BOSS 参数。每次扫描时，适配器只替换岗位关键词，保留其余已验证的原生筛选。它不得把行业、区域或其他原生条件翻译到另一个平台，也不得删除、近似映射或静默放宽任一用户手工设置的条件。

初始接入新平台时，继承模式优先于复杂的生成模式：即使平台的行业、融资阶段或更多筛选尚未建立通用映射，只要能可靠验证“同一搜索页、只替换关键词、其余筛选不变”，用户就可使用该平台。若页面状态无法可靠保存或关键筛选无法确认仍被保留，系统必须拒绝使用该模板并提示用户重新保存，不得猜测或复用 cookie、账号状态或隐藏页面状态。

同一份 Search Plan 可以为不同平台分别选择 `inherit`、`generated` 或 `disabled`：

- `inherit`：严格使用该平台用户预设的搜索模板；
- `generated`：只使用可精确映射的通用条件；
- `disabled`：本轮不扫描该平台。

界面必须显示每个平台使用的模式和已捕获筛选摘要。若继承模板与全局计划城市或薪资不同，应显式提示用户，而不是自行合并两套条件。

Search Plan 保存通用语义值，不保存 BOSS 原始代码：

- 区域以城市为键保存标准化区域名称。
- 公司规模保存受控的通用规模档。

平台适配器提供以下能力：

- 在指定城市读取区域和公司规模选项。
- 将通用区域和规模档精确映射为该平台的原生代码。
- 报告不支持或无法精确映射的条件。

原生目录缓存键必须包含平台和城市。目录的可用性不能从广州推断到上海或深圳。

## 执行流程

生成模式在每个城市开始扫描前：

1. 取得该平台、该城市的原生目录。
2. 校验 Search Plan 的区域与规模条件。
3. 将成功映射的原生代码与既有薪资、经验、求职类型、学历、关键词合并到搜索 URL。
4. 保存实际使用的通用条件、原生映射和目录版本到扫描快照，以便恢复时保持同一套条件。

任何城市的严格条件校验失败，都在发起该轮扫描前停止，不访问放宽条件后的搜索结果。

## 验收与测试

- 上海和深圳分别拥有独立的原生目录缓存与映射测试。
- 选中区域和规模时，生成 URL 包含该城市对应的原生代码。
- 未选择区域或规模时，不添加对应参数。
- 条件在某城市不支持或映射不精确时，扫描被拒绝且错误信息可定位到平台、城市和条件。
- 继承模式保留既有 URL 的区域、行业、规模等参数，且不被 Search Plan 覆盖。
- 既有薪资 lane、关键词、去重、低频串行与安全上限行为保持不变。
