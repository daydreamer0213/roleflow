# 多平台只读搜索与继承模板设计

## 状态

已确认基础方向，尚未实现。实现先建立多平台只读找岗骨架与每平台继承模板；真实智联或其他平台的 DOM 适配器必须在获得独立的只读页面证据后，另写实施计划。

## 用户目标

用户可以在 BOSS、智联及未来平台分别手工设置搜索条件，再让 RoleFlow 只替换岗位关键词并把各来源岗位放进同一个推荐队列。第一版只读搜索和岗位详情，不做任何平台的自动投递、聊天、私信读取或登录处理。

## 用户可见流程

用户先在某个平台设置好搜索页，例如：

~~~text
智联
广州 · AI/互联网 · 15–25k · 100–499 人 · 1–3 年
~~~

随后在 RoleFlow 中保存为该平台的模板。保存时系统展示“平台、筛选摘要、模板状态、保存时间”，用户确认后才可用于扫描。

同一份 Search Plan 可以并列配置：

~~~text
BOSS：继承 BOSS 模板
智联：继承智联模板
未来平台：关闭
~~~

每个平台可选三种模式：

| 模式 | 含义 |
|---|---|
| inherit | 严格使用用户在该平台保存的搜索模板，只替换岗位关键词 |
| generated | 只应用可精确映射的通用条件；初期仅区域和公司规模，其他条件按既有设计处理 |
| disabled | 本轮不扫描该平台 |

继承模板与全局计划的城市、薪资不同时，界面展示差异并等待用户决定；系统不自动合并或覆盖任一条件。

## 继承模式的严格规则

每份模板必须属于一个平台，保存以下不敏感信息：

- 平台标识；
- 规范化的搜索入口和允许保留的原生参数；
- 可见筛选摘要；
- 关键词替换位置或该平台的安全替换规则；
- 模板指纹、保存时间、最近验证状态。

不保存账号、cookie、完整聊天链接、验证码、凭据或未脱敏页面正文。

每次使用模板时，适配器必须确认：

1. 当前页面仍是该平台的有效搜索页；
2. 原筛选摘要或可验证原生参数仍与模板一致；
3. 唯一允许变化的是岗位关键词；
4. 替换后仍处于同一筛选上下文。

无法逐项确认时，该平台本轮必须停止并提示用户重新保存模板；不得复制隐藏浏览器状态、猜测参数、删除筛选或降低条件。

## 平台能力边界

每个平台适配器都要声明实际具备的能力，而不是假定 BOSS 功能普遍存在：

~~~text
扫描列表：是/否
读取详情：是/否
继承模板：是/否
生成条件：是/否
招聘方活跃度：有/无
直接投递：是/否
发起沟通：是/否
读取消息：是/否
~~~

第一期的目标能力只有：

~~~text
扫描列表：是
读取详情：是
继承模板：是
生成条件：按精确映射逐步开启
直接投递、沟通、消息：否
~~~

BOSS 的既有沟通能力继续留在 BOSS 专用路径，绝不因多平台重构被推广到其他平台。

## 统一岗位与来源保留

新平台将结果转换为现有统一岗位字段：source、sourceId、title、company、location、salary、experience、education、url、tags、description。平台特有字段只能以来源元数据或可选能力表达。

招聘方活跃度不是所有平台都有的信息。没有可靠活跃度的来源必须标为“该平台不可获取”，不能套用 BOSS 的三日活跃规则，也不能因此自动进入沟通路径。

跨平台同公司、同职位名的岗位只可标为“可能重复”，不能自动删除或合并。不同来源可能对应不同招聘方、更新时间或处理入口；用户决定保留哪一条。

## 架构边界

当前项目已经有 source/sourceId、site 运行记录、批次和站点锁等通用基础，但适配器工厂、搜索快照、城市编码、筛选、活跃度、报告命名和沟通清单仍有 BOSS 特定分支。

本期新增：

1. 平台注册表和能力声明；
2. 每平台搜索模板存储与 Search Plan 的平台列表；
3. 通用搜索快照入口，由适配器生成平台目标；
4. 串行的多平台只读扫描编排；
5. 一个假第二平台的离线适配器回归，用于证明架构不依赖 BOSS；
6. 来源可见、可能重复提示和按来源统计。

本期不实现真实智联页面选择器。真实平台接入的前置条件是：用户单独授权一次最小只读页面核对，获得不含敏感信息的 DOM fixture、稳定岗位标识、搜索模板验证证据和筛选保留证据。之后再写该平台专属实施计划。

## 安全与执行顺序

多平台扫描仍按平台串行执行：一个平台的扫描、详情读取、落库和检查点完成后，再进入下一平台。BOSS 继续使用它现有的站点锁、固定标签页、低频节奏与安全预算；新平台不能共享或削弱 BOSS 额度。

推荐顺序：

1. 用假第二平台完成平台注册表、模板和多站点扫描的离线回归；
2. 将 BOSS 迁移到新接口并证明行为不变；
3. 用户明确选择首个真实平台并授权最小只读探查；
4. 为该平台准备专属 fixture、适配器、回归和独立提交；
5. 只在真实读取稳定后，再评估是否需要该平台的投递或沟通能力。

## 验收

- 旧 BOSS Search Plan 无需用户重新配置，仍按原模式工作；
- 一个计划可保存 BOSS 继承模板与假第二平台继承模板；
- 每个平台只替换关键词，模板不匹配时仅停止该平台；
- 多平台扫描按来源分别建批次、保存检查点、显示结果；
- 假第二平台不具备沟通能力时，绝不进入 BOSS 沟通清单；
- 跨来源相似岗位保留两条记录并可显示可能重复；
- 全部测试仅使用临时数据库、假浏览器和 fixture；不访问真实招聘网站。
