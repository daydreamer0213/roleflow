# 双轮工作流验收说明

> 历史归档：本文记录 2026-07-20 的双轮方案，参数已经失效，不代表当前产品行为。当前每天最多三轮、条件式第三轮、两小时间隔和真实用量预算结算请查看 [`daily_workflow.md`](daily_workflow.md)。

## 当时产品路径

RoleFlow 每天最多手动执行两次完全相同的“一轮任务”，不区分上午或下午，也不会常驻运行。

```text
点击“执行一轮”
  -> 计算本轮目标、候选缺口和剩余安全预算
  -> 有足够有效库存时跳过扫描，否则选择未使用关键词
  -> 串行读取 BOSS 卡片和右栏详情
  -> 本地并发完成模型分析
  -> 展示本轮沟通清单，用户取消或保留勾选
  -> 用户确认后串行沟通
  -> 达到目标或清单耗尽后停止并保存结果
```

日目标为成功沟通 70 个，单轮最多 40 个。目标公式为：

```text
min(40, ceil(当天剩余目标 / 当天剩余轮次))
```

因此第一轮目标通常为 35。若第一轮成功 30，第二轮目标为 40；若第一轮成功 38，第二轮目标为 32。目标不是凑数要求，岗位供给不足时会按实际结果结束。

## 已完成的离线验收

运行：

```powershell
node tests/workflow_end_to_end_smoke.js
npm.cmd test
```

离线端到端测试使用本地 HTTP 工作台、SQLite、假扫描进程、假模型结果和假沟通适配器，不访问 BOSS。它验证：

- 同一天只能创建 sequence 1 和 sequence 2，第三轮返回稳定错误。
- 第一轮成功 30 后，第二轮目标自动调整为 40。
- 第二轮优先使用当天尚未使用的关键词。
- 已沟通、已失效、详情不完整的岗位不会再次进入默认清单。
- 达到成功目标后，未访问的替补项以零点击停止，并保留为后续库存。
- 重启只校准失联状态，不会自动继续浏览器访问。
- 待审核清单、批次关联和阶段指标能够从数据库恢复。
- 日志和审计事件可通过 workflow、scan、scan batch、communication batch 四类 ID 串联。

## 日常操作

1. 启动 RoleFlow，确认模型连接正常，并在 Edge 打开已登录的 BOSS 页面。
2. 在“今日求职任务”点击“执行一轮”。
3. 扫描和模型分析完成后，检查默认勾选清单；取消不想沟通的岗位。
4. 点击“确认清单”，再次检查批次后点击“开始沟通”。
5. 本轮结束后查看成功数、失效数和缺口原因；当天需要时再执行第二轮。
6. 中途退出后返回同一轮，使用“继续本轮”；不要创建补充第三轮。

诊断页展示最近脱敏日志，完整 JSONL 位于 `.runtime\logs`。排错时优先记录页面错误码和 `workflowRunId`，再关联 `scanRunId`、`scanBatchId` 或 `communicationBatchId`。

## 低流量真实页面验收

离线通过不等于真实页面已经完整通过。首次上线校验按以下顺序执行，所有步骤串行：

1. 确认 BOSS 正常登录、没有安全验证或访问限制，且 RoleFlow 没有其他扫描或沟通任务。
2. 采集验收只使用 1 个关键词，最多 10 张卡片、3 个右栏详情。
3. 采集验收阶段不执行自动沟通；先检查岗位身份、卡片字段、右栏正文、活跃状态和日志。
4. 采集证据确认后，另选 1 个未投且仍在招聘的岗位建立沟通清单。
5. 用户确认岗位和公司一致后，只执行 1 次沟通；核对按钮变化、聊天对象和数据库结果。
6. 任何登录失效、安全验证、异常重定向、页面结构漂移、岗位身份不一致或结果不明确，立即停止当天验收，不自动重试。

采集校准可从高级扫描入口执行，或使用下面的 broad 校准命令；将 `<Plan ID>` 和 `<关键词>` 替换为当前方案值：

```powershell
node src/cli.js scan --plan <Plan ID> --browser edge --scan-mode broad --keywords "<关键词>" --max-cards 10 --max-detail-total 3 --browser-page-budget 20
```

该命令只用于真实页面结构校准，不计入当天两轮正式工作流，也不会执行沟通。

真实校验完成后，先执行一轮小规模正式任务并检查转化率；没有必要为了凑够目标追加第三轮或扩大当天预算。

## 恢复边界

- `review_required`：无需浏览器进程，可以关闭程序后再回来确认。
- `interrupted`：保留原 sequence、关键词、预算和批次；继续时不会占用第三个槽位。
- 扫描或沟通子任务超过心跳窗口才会被判定为失联；新鲜任务不会被重启逻辑中断。
- 恢复过程只修改本地状态，绝不自动导航、刷新、点击或发送消息。
